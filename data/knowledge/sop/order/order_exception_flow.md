# 售中订单异常处理 SOP

## 适用场景

适用于售中订单中的复杂、异常或需要跨系统核验的问题。

## 标准处理流程

1. 先确认订单号、支付状态、发货状态和用户诉求。
2. 未支付订单按普通取消或重新下单处理；已支付订单需区分待审核、待出库、已发货。
3. 重复扣款、风控拦截、企业订单审批、恢复关闭订单必须走人工复核。
4. 涉及资金时不得承诺退款必成，只能说明核实路径和预计反馈时间。

## 必须升级的情况

- 涉及资金、赔付、风控、质检争议、投诉或高价值订单。
- 用户明确要求人工、主管、投诉或监管渠道。
- 知识库政策与系统状态不一致。
